# MedMinder Concierge ADK/MCP Package

MedMinder Concierge is a safety-first multi-agent medication routine assistant for elderly users and caregivers.

This package is the executable companion to the Kaggle notebook. It does not only describe the agents: it runs the same deterministic workflow through safety screening, schedule lookup, package verification, self-confirmation, escalation, and history tracking.

## One-command demo

From this folder:

```bash
python demo_runner.py --scenario success
python demo_runner.py --scenario wrong-package
python demo_runner.py --scenario loose-pill
python demo_runner.py --scenario unsafe-dose
python demo_runner.py --scenario no-confirmation
```

Each run prints the agent-by-agent execution trace and saves:

```text
artifacts/demo/latest_demo_trace.json
```

The trace is generated by `medminder_agent/core_workflow.py`, not by a separate presentation script.

## Real caregiver notification demo

For a real phone notification, install the ntfy app on a phone and subscribe to:

```text
medminder-concierge-demo
```

Then run an escalation scenario with notification enabled:

```bash
python demo_runner.py --scenario wrong-package --notify --ntfy-topic medminder-concierge-demo
```

Without `--notify`, MedMinder builds the exact alert payload but keeps it in dry-run mode.

## Phone-style caregiver dashboard

Run the local dashboard:

```bash
python caregiver_dashboard.py
```

Open:

```text
http://127.0.0.1:8765
```

The dashboard executes the real workflow, shows the agent trace, writes `artifacts/dashboard/latest_dashboard_trace.json`, and can send a real ntfy caregiver alert for escalation cases.

Deployment notes are in:

## Optional live Gemini/ADK path

`medminder_agent/agent.py` exposes the workflow functions as ADK-style tools on `root_agent`. If Google ADK and Gemini credentials are configured, this can be run with:

```bash
adk run medminder_agent
```

The deterministic workflow and notification path remain runnable without API keys.

## Demo scenarios

- `success`: scheduled medicine, original package text matches, user confirms completion.
- `wrong-package`: package text shows a different medicine, so caregiver escalation is created.
- `loose-pill`: loose pill identification is blocked and escalated.
- `unsafe-dose`: dosage-change request is blocked before schedule/package tools run.
- `no-confirmation`: package matches, but the user does not confirm completion, so the caregiver is alerted.

## Safety

This demo uses mock data only.

It does not give medical advice, change dosages, identify loose pills, prove ingestion, send real SMS/calls, or include API keys.
